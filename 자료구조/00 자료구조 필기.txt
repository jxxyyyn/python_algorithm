자료구조: 데이터를 저장하고 관리하기 위해 사용하는 구조
        데이터의 효율적인 접근 및 조작을 가능하게 해주는 관리 방식
        모든 경우에 제일 좋은 자료 구조는 없음

스토리지 - 데이터가 영구적으로 저장되는 곳
메모리 - 데이터를 임시로 저장하는 곳 (데이터 저장하고 받아오는 게 빠르다)
  -> 데이터를 메모리에서 잘 사용하도록 하기 위해서 자료구조 활용
  -> 일정한 칸으로 나뉘어 있고, 각 칸에는 데이터를 저장할 수 있다. 자신만의 주소 가짐

RAM: 임의 접근 메모리, 저장 위치를 알면 접근할 때 일정한 시간이 걸림
     시간 복잡도 O(1)
     한 칸이 저장할 수 있는 가장 기본적인 용량의 단위가 byte

레퍼런스 : 데이터에 접근할 수 있게 해주는 값, 주소보다 조금 더 포괄적인 표현
x=95일 때 정수 95가 담겨 있는 것이 아닌 레퍼런스가 담겨 있음
하지만 실제로 연산 등에서 변수를 사용할 때는 저장된 값을 알아서 받아옴
id 함수 - 저장된 데이터의 메모리주소를 정수로 표현한 값 알아내는 함수, 똑같은 주소에 저장되어 있는 데이터는 똑같은 데이터임
